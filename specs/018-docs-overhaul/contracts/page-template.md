# Contract: Documentation Page Template

Every documentation page must follow this minimal structure to satisfy FR-011.

## Required sections

- H1 title
- Purpose/Overview (1-2 paragraphs)
- Prerequisites
- Content body (topic-specific)
- Related
- Next steps

## Optional sections

- Applies to (required when behavior differs across releases)
- Tested with (recommended alongside Applies to)
- Troubleshooting (recommended for setup/config pages)

## Link rules

- Prefer repository-relative links.
- Avoid bare URLs when a named link is clearer.
- When referencing commands, include copy/paste-ready snippets.
